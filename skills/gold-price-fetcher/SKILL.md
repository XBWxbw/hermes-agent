# 黄金价格获取技能

通过 **HTML + jin.js 双源方案** 直接抓取 [huangjinjiage.cn](http://www.huangjinjiage.cn)，无需任何浏览器工具，零额外依赖（仅标准库 + 可选 beautifulsoup4）。

## 数据来源

| 数据源 | URL | 内容 | 方式 |
|--------|-----|------|------|
| jin.js 实时接口 | `http://res.huangjinjiage.com.cn/jin.js` | 国际/国内实时价格（含最高/最低价）| `urllib` 直接请求 |
| 首页 HTML | `http://www.huangjinjiage.cn` | 品牌金店报价 + 行情动态 | `urllib` + BeautifulSoup |

**两个接口均直接返回数据，无 JS 动态渲染，无需浏览器。**

## 前置依赖

```bash
# 可选：安装 beautifulsoup4 提升品牌/动态解析质量
pip install beautifulsoup4

# requests 为可选加速（不安装时自动用 urllib 替代）
pip install requests
```

> `parse_gold.py` 在只有 Python 标准库时也能运行，但安装 `beautifulsoup4` 可以获得更准确的品牌/动态解析结果。

## 执行流程

### Step 1：运行解析脚本

直接通过 `terminal` 调用 `parse_gold.py`，脚本内部自动完成双源抓取：

```bash
# 输出 JSON（默认）
python scripts/parse_gold.py --output result.json

# 输出 Markdown
python scripts/parse_gold.py --format markdown --output result.md

# 离线模式（从已有 HTML 文件解析，不请求网络）
python scripts/parse_gold.py --input huangjinjiage.html --format markdown
```

**在 Agent 内的标准调用方式**（使用绝对路径）：

```python
# 方式 A：CLI
python "E:\Skills\gold-price-fetcher\scripts\parse_gold.py" --format markdown --output "$env:TEMP\gold_result.md"

# 方式 B：Python import
from scripts.parse_gold import fetch_and_parse

data = fetch_and_parse()             # 返回 dict
md   = fetch_and_parse(fmt="markdown")  # 返回 Markdown 字符串
```

### Step 2：整理数据

解析结果包含四个板块：

1. **国际贵金属报价**（来自 jin.js）
   - 国际金价、国际银价、国际铂金、国际钯金
   - 字段：最新价、涨跌、幅度、**最高价**、**最低价**、昨收价（均为实时准确值）

2. **国内黄金报价**（来自 jin.js）
   - 黄金延期（元/克）、白银延期（元/千克）
   - 附：沪金99、M黄金延期（上金所现货）
   - 字段同上，含最高/最低价

3. **品牌金店今日金价**（来自 HTML）
   - 内地品牌（元/克）：周大福/周生生/老凤祥/潮宏基等11家
   - 香港品牌（港币/两）：周大福/周生生/谢瑞麟等
   - 字段：黄金价格、铂金价格、金条价格

4. **行情动态**（来自 HTML）
   - 最新 3 条，含时间戳和正文

## 注意事项

- **无浏览器依赖**：完全通过 HTTP 请求获取数据，无需 agent-browser / Playwright / Chrome
- **jin.js 数据最准确**：国际/国内价格含实时最高/最低价，是网站 JS 动态渲染的真实数据源
- **品牌金价每日更新一次**（早上9-10点），并非实时涨跌
- 国际金价单位为美元/盎司，国内延期单位为元/克
- 汇率未知时用 7.2 估算，并注明"估算汇率"
- `beautifulsoup4` 未安装时自动回退到正则解析，品牌/动态解析质量略低但仍可用
- `requests` 未安装时自动用 `urllib` 替代，无功能差异
